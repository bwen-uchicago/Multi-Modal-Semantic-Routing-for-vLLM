# Multi-Modal Semantic Routing for vLLM: Accent-Aware Speech Recognition via TTS-Synthesized Training Data

[![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg)](https://opensource.org/licenses/Apache-2.0)
[![NAIRR](https://img.shields.io/badge/NAIRR-Core%20AI%20Track-blue)](https://nairrpilot.org/)

## Overview

Current speech-to-text (STT) systems can reliably identify *which language* is being spoken—English, Mandarin, Spanish, etc.—but struggle to recognize the **accent** of the speaker within a given language. A Mandarin-native speaker, a Russian-native speaker, and a Hindi-native speaker may all be speaking English, yet their pronunciation patterns differ systematically. These accent-specific variations lead to significantly higher word error rates (WER) compared to native speakers, degrading the quality of downstream applications such as voice assistants, real-time translation, and conversational AI.

This project introduces a framework for **accent classification and accent-adaptive speech recognition**, leveraging synthetic training data generated by multilingual text-to-speech (TTS) models. By routing recognized accents to specialized model adaptations (e.g., LoRA fine-tuned STT modules), we enable a semantic routing layer that dynamically optimizes transcription quality per speaker—a form of *multi-modal semantic routing* for voice-driven LLM serving pipelines.

## Problem Statement

### The Accent Gap in Speech Recognition

Modern ASR systems are trained predominantly on native-speaker corpora. When non-native speakers interact with these systems, WER increases substantially:

- **Mandarin-accented English** speakers exhibit distinct tonal carryover and consonant substitution patterns
- **Russian-accented English** speakers show characteristic vowel reduction and stress pattern differences
- **Hindi-accented English** speakers demonstrate retroflex consonant transfers and syllable-timed rhythm
- **Spanish-accented English** speakers display vowel insertion and voicing pattern differences

These systematic deviations are **predictable and classifiable**, yet current STT pipelines treat all English input identically, leaving significant accuracy gains on the table.

### Why Accent Classification Matters

Accurate accent identification unlocks two powerful optimization paths:

1. **Accent-Adaptive Transcription**: Route accented speech to specialized LoRA adapters fine-tuned for specific accent profiles, reducing WER without retraining the base model.
2. **Language Switching**: Proactively suggest that non-native speakers switch to their native language for the interaction. Modern voice AI systems can seamlessly swap STT and TTS models across languages, often yielding dramatically better recognition accuracy and user experience.

## Approach: TTS-Synthesized Accent Training Data

### The Data Collection Challenge

Building accent classifiers and accent-adapted STT models traditionally requires large corpora of accented speech with accurate transcriptions—a costly and labor-intensive process. The fundamental paradox: STT systems perform poorly on accented speech (that's the problem we're solving), so using STT to auto-transcribe accented recordings produces unreliable labels. Manual transcription at scale is prohibitively expensive.

### Our Solution: Synthetic Data Generation

We observe that modern multilingual TTS models inherently capture accent characteristics. When a voice model trained on a particular language is prompted to speak English text, the resulting audio naturally exhibits accented English pronunciation patterns. This insight enables a scalable data generation pipeline:

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────────┐
│  English     │────▶│  Multilingual    │────▶│  Accented English   │
│  Text Corpus │     │  TTS Models      │     │  Audio + Labels     │
└─────────────┘     │  (per language)   │     │  (ground truth text │
                    └──────────────────┘     │   already known)    │
                                             └─────────────────────┘
```

**Key advantages of this approach:**

- **Free ground-truth transcriptions**: Since we control the input text, the transcript is known *a priori*—no labeling required.
- **Scalable accent diversity**: The same text can be synthesized across dozens of accent profiles, producing balanced training sets effortlessly.
- **Controllable conditions**: We can vary speaking rate, noise conditions, and accent intensity systematically.
- **Cost efficiency**: Generating thousands of hours of accented speech requires only compute, not human annotators.

## Research Plan

### Phase 1: Data Generation Pipeline
- Curate a diverse English text corpus (conversational, command, mixed-domain)
- Select and benchmark multilingual TTS models for accent fidelity
- Generate synthetic accented speech datasets across target accent groups:
  - Mandarin-accented English
  - Russian-accented English
  - Hindi-accented English
  - Spanish-accented English
  - Native American English (control)
- Validate synthetic accent quality against real accented speech samples

### Phase 2: Accent Classifier
- Extract acoustic features (MFCCs, Mel spectrograms, wav2vec2 embeddings)
- Train accent classification models (CNN, Transformer-based)
- Evaluate on both synthetic and real accented speech
- Analyze the transferability of synthetic-trained classifiers to real-world accented speech

### Phase 3: Accent-Adaptive STT
- Fine-tune STT models (Whisper) with accent-specific LoRA adapters
- Build a semantic routing layer that chains: accent classification → adapter selection → transcription
- Measure WER improvements per accent group vs. baseline Whisper
- Benchmark end-to-end latency of the routing pipeline

### Phase 4: Integration with vLLM Serving
- Implement the accent-aware routing as a preprocessing module for vLLM-based voice pipelines
- Design the multi-modal routing API for real-time accent detection and model adaptation
- Evaluate system-level metrics: throughput, latency, and accuracy under concurrent multi-accent workloads

### Prerequisites
- Python 3.10+
- PyTorch 2.0+
- CUDA-capable GPU (recommended: A100 or H100)
